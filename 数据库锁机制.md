一、表锁 VS 行锁
1. 表锁（Table Lock）
- 锁整张表
- 粒度大，并发小
- 开销小，不会死锁
- MyISAM默认用表锁

2. 行锁（Row Lock）
- 锁一行/几行
- 粒度小，并发高
- 开销大，可能死锁
- InnoDB支持
- 必须走索引，不然会退化成表锁

行锁解决什么问题：
- 多个事务同时修改同一行时，只有一个能改
- 避免脏写、覆盖更新


二、悲观锁 VS 乐观锁
1. 悲观锁（Pessimistic）
- 认为一定会冲突
- 先加锁，再操作
- 实现：sql SELECT * FROM user WHERE id=1 FOR UPDATE;
- 使用：写多、并发高、冲突频繁

2. 乐观锁（Optimistic）
- 认为冲突很少
- 不加锁，更新时比对版本
- 实现：sql UPDATE user SET...,version=version+1 WHERE id=1 AND version=旧版本
- 影响行数=0说明冲突，需要重试
- 适合：读多写少


三、常见问题
1. InnoDB行锁怎么加的？
- 通过索引项加锁
- SQL没走索引->锁全表

2. 乐观锁怎么实现？
- 加版本号/时间戳
- 更新时比对版本号，一致才更新

3. 行锁 VS 表锁怎么选择？
- 高并发、只改少量行->行锁
- 批量操作、改整张表->表锁
