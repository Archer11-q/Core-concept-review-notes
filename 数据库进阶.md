一、ACID
1. 原子性：
  - 事物要么全部成功，要么全部失败，回滚到原来样子
  - 实现：undo log
2. 一致性：
  - 执行前后，数据库的完整性约束不被破坏
  - 是最终目的，由其他三个特征保证
3. 隔离性：
  - 多个事务同时跑，互相看不见中间状态
  - 实现：锁+MVCC（多版本并发控制）
4. 持久性：
  - 一旦提交，永久生效，宕机也不丢
  - 实现：redo log


二、三种问题 & 四大隔离级别
三种问题：
1. 脏读：读到别人未提交的数据
2. 不可重复读：同一事务内，两次读同一行结果不一样
3. 幻读：同一事务内，两次查询的行数不一样

四大隔离级别：
1. 读未提交：什么都解决不了，会脏读
2. 读已提交：解决脏读
3. 可重复读：解决脏读+不可重复读
   - 一个事务内，看到的数据始终一样
   - MySQL InnoDB默认级别
4. 串行化：全解决，但性能最差，相当于排队执行


三、数据库锁
1. 行锁
- 锁一行
- 粒度小，并发高
- InnoDB常用
2. 表锁
- 锁整张表
- 粒度大，并发低
- MyISAM默认
3. 悲观锁
- 认为一定会冲突
- 先加锁再操作
- select...for update
4. 乐观锁
- 认为冲突很少
- 不加锁，更新时比对版本号
- 版本号/时间戳机制
